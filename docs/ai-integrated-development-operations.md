# AI統合開発運営システム

## 最終目的

AI統合開発グループは、以下をAI主体で構築・運営する。

1. 会計・財務アプリ開発
2. ゲーム開発
3. YouTube/TikTokショート運営
4. AI会計事務所構築
5. 自動化業務システム構築

## 基本方針

- GitHubを唯一の開発管理基盤とする
- Notionを経営、仕様、議事録基盤とする
- すべてIssueベースで管理する
- 必ずPRレビューを実施する
- タスクを小分けにする
- 不明点は推測せずIssue化する
- 秘密情報は絶対に出力しない
- token、password、API keyは出力しない
- 法律、著作権、規約違反は禁止する

## AI役割

### Claude

- 実装責任者
- アーキテクト
- リファクタ
- DB設計

### Codex

- GitHub補助

### ChatGPT

- 業務設計
- 品質管理
- ドキュメント
- UI改善
- プロンプト管理

### Gemini

- 市場調査
- Steam分析
- YouTube分析
- API仕様調査
- 競合分析

## 共通ルール

- 1Issue=1目的
- Issueに背景を書く
- PRには目的を書く
- 変更理由を書く
- レビュー結果を書く
- 例外対応を書く

## GitHub運営ルール

### Issueテンプレに含めるもの

- 背景
- 目的
- 対応内容
- 完了条件
- 禁止事項

### PRテンプレに含めるもの

- 変更概要
- 影響範囲
- テスト結果
- 未解決事項
- レビュー内容

## Notion運営ルール

Notionでは以下のDBを管理する。

- タスク管理
- バグ管理
- アイデア管理
- 動画ネタ管理
- ゲーム仕様
- API仕様
- 顧客要望

## 開発思想

- MVPを最優先する
- 完璧主義を避ける
- 小さく早く改善する
- 再利用性を重視する
- 自動化を優先する
- AI同士の役割衝突を避ける
