# 資金調達・CF可視化アプリ MVP 最終資料案

この資料は、MVPの最終提出・営業デモ・社内共有で必要になる成果物のたたき台です。

## 1. アプリで生成する資料

### 銀行向け借入資料

- 借入目的
- 会社概要・事業状況
- 直近月次CF
- 資金使途
- 返済原資
- 既存借入・返済状況
- リスクと対応策
- 補足資料一覧

銀行向けは、返済原資・既存借入・入出金の安定性を重視します。数字はMF同期データから引用し、不足情報は「要確認」として残します。

### 日本政策金融公庫向け資料

- 事業概要
- 借入目的
- 資金使途
- 事業の継続性・成長性
- 返済計画の前提
- 雇用・地域性・創業背景などの補足
- リスクと改善策

公庫向けは、事業背景・資金の必要性・継続性をやや厚めにします。創業・新規投資の場合は自己資金、経験、取引見込みを追記します。

### 社内稟議書

- 申請概要
- 希望借入条件
- 資金使途
- 借入しない場合のリスク
- 代替案
- CFへの影響
- 決裁事項

社内向けは、決裁者が判断できるように、結論・金額・期限・リスクを冒頭で確認できる構成にします。

## 2. デモで必要な資料

- 画面遷移図: ログイン → 会社作成 → MF連携 → 同期 → CF確認 → 資料生成
- DB構成図: company_id によるデータ分離、mf_connections、journals、classification、documents
- 同期仕様メモ: 初回同期、手動同期、当月+前月取得、履歴保存
- セキュリティ説明: token暗号化、環境変数、ログマスク、company単位分離
- MVP対象外リスト: 予測分析、複雑な承認、金融機関別最適帳票、自動学習分類

## 3. 追加アイデア

- 要確認仕訳を資料生成前に警告表示する
- 借入希望額に対して、直近CFから「説明に使える材料」と「不足材料」を自動列挙する
- 公庫向けには「創業/設備/運転資金」モードを追加する
- 銀行向けには「既存借入一覧」入力欄を追加する
- 社内稟議向けには「借入しない場合の資金ショート時期」を手入力で補足できる欄を追加する
- PDF出力前に「要確認」マーカーを一覧表示する

## 4. 第4週の安定化チェック

- OAuth期限切れ時の再連携導線
- 手動同期失敗時の履歴保存
- 仕訳取得のページング
- company_id の取り違えがないこと
- 生成文が数字を創作していないこと
- PDFで日本語が崩れないこと
- ログにトークンが出ていないこと
